Vẽ một hình lập phương

Trong blog này, chúng ta sẽ cùng nhau vẽ được một hình lập phương (cube) lên canvas 😳. Trước khi vẽ được 1 cục cube lên màn hình, ta cần có thông tin của cube đó trước, là tập hợp tất cả các tam giác tạo nên nó. Ta đã làm được một phần việc này ở phần 2 của blog trước, nhưng ta đang mặc định tâm của cube đó ở gốc tọa độ (0, 0, 0) và có kích thước là 1. Ta cần tạo một hàm để tạo ra cube ở vị trí và kích thước bất kỳ, hàm đó sẽ như sau (mình nghĩ là không cần giải thích):

Tuy nhiên, hàm này khá khó đọc và hiểu (và cũng khó bảo trì về sau), vì sử dụng javascript object lung tung mà không rõ ràng chúng là gì. Cho nên trước đó ta cần định nghĩa kiểu dữ liệu cho điểm và tam giác, trước tiên là điểm, mình đặt tên là Vector3 cho giống Unity, gồm các hàm cơ bản như cộng, trừ, nhân, chia, nhân vector, tích vô hướng, tích có hướng, tính độ dài và chuẩn hóa về vector đơn vị: 
class Vector3 {
    constructor(x = 0, y = 0, z = 0) {
        this.x = x;
        this.y = y;
        this.z = z;
    }

    static add(vec1, vec2) {
        return new Vector3(vec1.x + vec2.x, vec1.y + vec2.y, vec1.z + vec2.z);
    }
    static sub(vec1, vec2) {
        return new Vector3(vec1.x - vec2.x, vec1.y - vec2.y, vec1.z - vec2.z);
    }
    static mul(vec, value) {
        return new Vector3(vec.x * value, vec.y * value, vec.z * value);
    }
    static div(vec, value) {
        return new Vector3(vec.x / value, vec.y / value, vec.z / value);
    }
    static scale(vec1, vec2) {
        return new Vector3(vec1.x * vec2.x, vec1.y * vec2.y, vec1.z * vec2.z);
    }
    static dot(v1, v2) {
        return v1.x * v2.x + v1.y * v2.y + v1.z * v2.z;
    }
    static cross(v1, v2) {
        let v = new Vector3(
                v1.y * v2.z - v1.z * v2.y,
                v1.z * v2.x - v1.x * v2.z,
                v1.x * v2.y - v1.y * v2.x);
        return v;
    }
	magnitude(){
		return Math.hypot(this.x, this.y, this.z);
    }
    normalize() {
        let length = this.magnitude();
        if (length === 0)
            return this;
        this.x /= length;
        this.y /= length;
        this.z /= length;
        return this;
    }
}

class Triangle {
    constructor(vertices = []) {
        if (vertices.length == 0) {
            this.vertices = [];
            for (let i = 0; i < 3; i++) {
                this.vertices.push(new Vector3(0, 0, 0));
            }
        } else {
            this.vertices = vertices;
        }
    }
}

Có được hàm makeCube rồi, ta sẽ tiến hành vẽ, quay lại với canvas mà đã được setup ở phần 1, ta tạo một cube ở vị trí (0, 0, 2) (chính giữa màn hình và cách màn hình 2 đơn vị) và có kích thước là (0.7, 0.7, 0.7): 
let cube1 = makeCube(new Vector3(0, 0, 2), new Vector3(0.7, 0.7, 0.7));
Sau đó đối với từng tam giác của cube1, ta sẽ vẽ 3 đường nối 3 đỉnh của nó lại với nhau, sử dụng các hàm của canvas 2D context: 
for (let tri of cube1) {
    vertices = tri.vertices;
    ctx.beginPath();
    ctx.moveTo(vertices[0].x, vertices[0].y);
    ctx.lineTo(vertices[1].x, vertices[1].y);
    ctx.lineTo(vertices[2].x, vertices[2].y);
    ctx.closePath();
    ctx.stroke();
}
Kết quả ra như sau:
Khoan đã, sao lại không có gì hết nhỉ? Thật ra là có, nếu bạn để ý bạn sẽ thấy một dấu chấm nhỏ ở góc trái trên của canvas, đó chính là cube1 của mình, nó nhỏ như vậy vì mình đang cố vẽ một hình lập phương kích thước 0.7 trong 1 canvas 400x300. Cho nên trước khi vẽ các tam giác, ta nên chuẩn hóa từng đỉnh của nó về lại khoảng hợp lý của canvas, bằng cách nhân kích thước canvas và cộng một nửa kích thước canvas để đưa nó vào giữa. Tuy nhiên, ta nên biết rằng HTML Canvas có y hướng xuống (càng xuống càng tăng), nhưng không gian 3D của chúng ta lại có y hướng lên, cho nên ta phải đổi dấu của y trước khi nhân, như sau: 
for (let tri of cube1) {
    vertices = tri.vertices;
    for (let i = 0; i < vertices.length; i++) {
        vertices[i] = new Vector3(
            vertices[i].x * canvas.width + canvas.width / 2,
            -vertices[i].y * canvas.height + canvas.height / 2,
            vertices[i].z);
    }
    ctx.beginPath();
    ctx.moveTo(vertices[0].x, vertices[0].y);
    ctx.lineTo(vertices[1].x, vertices[1].y);
    ctx.lineTo(vertices[2].x, vertices[2].y);
    ctx.closePath();
    ctx.stroke();
}
Nó sẽ vẽ ra như sau:
Khoan đã, tại sao:
  1. Nó lại là hình chữ nhật chứ không phải hình vuông?
  2. Nó lại phẳng lì 2D như thế mà không phải 3D?
  3. Lại có hai đường chéo đó?
Về ý thứ nhất, đó là vì canvas cũng là hình chữ nhật, mình chỉ đang nhân đúng kích thước của nó, do đó mình phải tự sửa lại với tỉ lệ (aspect ratio) của canvas. Chỉ đơn giản là lấy x chia cho chiều rộng và nhân chiều cao canvas trước khi làm các phép tính khác: $$ (x * \frac{h}{w}) * w + \frac{w}{2} = x * h + \frac{w}{2} $$ Code sẽ thành: 
vertices[i] = new Vector3(
    vertices[i].x * canvas.height + canvas.width / 2,
    -vertices[i].y * canvas.height + canvas.height / 2,
    vertices[i].z);
Thật ra nếu bạn giữ nguyên x và lấy y chia chiều cao nhân chiều rộng nó cũng sẽ ra được hình vuông, nhưng mà kích thước nó sẽ lớn hơn, và thêm hậu quả là khi vẽ trên canvas dài nó sẽ bự hơn rất nhiều nữa, tính theo x nó sẽ ổn định trên mọi loại resolution màn hình thông dụng.

Về ý thứ hai, lý do mình chỉ thấy hình chữ nhật 2D mà không phải hình cube 3D là vì mình đang chiếu tất cả điểm thẳng băng tới màn hình, nói cách khác là đang sử dụng vị trí trong không gian để vẽ luôn tọa độ trên màn hình, cái này được gọi là phép chiếu Orthographic, cái này như cách bạn hay vẽ các loại hình chiếu ở môn Công nghệ lớp 10. Còn đối với ngoài đời, bạn sẽ nhìn mọi thứ theo kiểu "càng gần càng to", đó là phép chiếu Perspective.

Về ý thứ ba, hai đường chéo đó chỉ là một trong những đường tạo nên những tam giác mà tạo nên cube đó, mình đang vẽ hết tất cả các đường, nên mình thấy cả mặt FRONT lẫn mặt BACK.

Quay về Perspective Projection, cài đặt nó rất đơn giản, bạn chỉ cần dùng tới 1 giá trị mà có thể bạn đã quên khi đang theo dõi mình làm, đó chính là tọa độ z (chính là khoảng cách từ màn hình tới cube, đối với trường hợp hiện tại). Mình chỉ cần chia cả x và y cho z là sẽ tạo ra cảm giác "càng gần càng to": 
for (let tri of cube1) {
    vertices = tri.vertices;
    for (let i = 0; i < vertices.length; i++) {
        let v = Vector3.div(vertices[i], vertices[i].z);    // <--------------
        vertices[i] = new Vector3(
            v.x * canvas.height + canvas.width / 2,
            -v.y * canvas.height + canvas.height / 2,
            1);
    }
    ctx.beginPath();
    ctx.moveTo(vertices[0].x, vertices[0].y);
    ctx.lineTo(vertices[1].x, vertices[1].y);
    ctx.lineTo(vertices[2].x, vertices[2].y);
    ctx.closePath();
    ctx.stroke();
}

Như vậy, chúng ta đã vẽ thành công một cube lên canvas, lúc này mình mới thấy hết cả đường tạo nên các tam giác mà tạo nên cube, việc mình thấy hết tam giác như vậy được gọi là cách hiển thị Wireframe, mình sẽ chỉ cách làm mất chúng sau.

let cube1 = makeCube(new Vector3(0.6, -0.7, 2.5), new Vector3(0.7, 0.7, 0.7));
